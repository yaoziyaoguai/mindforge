## Changes

### U1: vitest + happy-dom Infrastructure

**Files:**
- `web/package.json` — 新增 devDependencies: vitest, happy-dom, @testing-library/react, @testing-library/jest-dom; 新增 `"test": "vitest run"` script
- `web/vite.config.ts` — 新增 `test` 配置块 (environment: happy-dom, include: src/**/*.test.{ts,tsx}, setupFiles)
- `web/src/test/setup.ts` — 引入 @testing-library/jest-dom/vitest matchers

**Dependency justification:**
- vitest (~3MB): Vite 原生测试框架，零配置 TSX 支持
- happy-dom (~2MB): 轻量 DOM 实现，组件测试 environment
- @testing-library/react (~500KB): React 组件测试标准库
- @testing-library/jest-dom (~100KB): 语义化 DOM matchers

全部 devDependencies，不污染生产构建。

### U2: Example Component Test

**Files:**
- `web/src/components/__tests__/ErrorState.test.tsx` — 2 个测试用例，验证 ErrorState 组件渲染

选 ErrorState 因为它是最简单的纯展示组件：无副作用、无 hooks、无 context、无 API 调用。

### U3: Python Coverage Configuration

**Files:**
- `pyproject.toml` — 新增 `[tool.coverage.run]` (source + omit) 和 `[tool.coverage.report]` (show_missing)

pytest-cov 已在 dev dependencies 中，仅缺配置。不设 fail_under 阈值 — 先建立可观测性。

### U4: web_config_service.py — Extract Env Detection

**Files:**
- `src/mindforge_web/services/web_config_env.py` — 新建，包含 DotenvPresence, find_dotenv_file(), parse_env_key(), read_dotenv_presence(), env_present()
- `src/mindforge_web/services/web_config_service.py` — 移除 DotenvPresence 定义 + 3 个方法实现（改为委托），移除未使用的 `dataclasses` import

提取范围：纯函数 env 检测逻辑。WebConfigService 中的 `_env_present` 等实例方法保留（依赖 self.cfg 状态），不在此次提取范围内。
