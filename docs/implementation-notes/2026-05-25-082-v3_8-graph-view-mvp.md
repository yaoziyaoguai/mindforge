## 实现范围

v3.8 在 v3.7 graph ontology 基础上构建了图视图 MVP。核心变化：

### 1. API Schema 同步到 v3.7 Ontology

**`src/mindforge_web/schemas.py`:**
- `GraphNodeResponse.type`: `"card"|"source"|"wiki_section"|"tag"|"concept"` → `"card"|"source"|"wiki_section"|"tag"|"community"|"topic"|"entity"|"concept_candidate"`
- `GraphEdgeResponse.edge_type`: 移除 `mentions`, `approval_state_of`；新增 `has_tag`, `in_section`, `contains`, `includes`, `belongs_to_topic`, `mentions_candidate`, `resolves_to`
- `GraphResponse.center_type`: 同步更新

**`web/src/api/types.ts`:**
- `GraphNodeType` 和 `GraphEdgeType` 同步更新

### 2. GraphView 页面（vis-network 可视化）

**新文件:**
- `web/src/pages/GraphPage.tsx`: 图浏览页面，支持 8 种 NodeType 的图探索
- `web/src/components/GraphCanvas.tsx`: vis-network 渲染组件，含右侧 info panel

**vis-network 依赖:**
- `npm install vis-network` (v10.1.0, standalone/peer ESM)
- 使用 `vis-network/standalone` ESM 导入
- ForceAtlas2 物理引擎布局
- 不同 NodeType 使用不同的颜色/形状：
  - card: 靛蓝 box, source: 蓝色 database, wiki_section: 紫色 square
  - tag: 橙色 triangle, community: 绿色 dot, topic: 红色 star
  - entity: 粉色 diamond, concept_candidate: 灰色 dot

**交互:**
- 拖拽/缩放/框选
- 点击节点 → 右侧 info panel 展示 ID/type/label/card_count
- 点击边 → 右侧 info panel 展示 evidence/reason/strength
- 点击 card/source/tag 节点 → 以该节点为中心重新探索（图导航）
- 支持 1/2/3-hop depth 切换

### 3. 导航入口

**侧边栏:**
- `web/src/components/Sidebar.tsx`: 新增"知识图谱"导航项（GitBranch 图标）

**Card detail:**
- `web/src/components/GraphNavigationPanel.tsx`: header 新增"查看图谱"按钮，
  可跳转到 `/graph?card=<cardId>` 的全屏图视图
- GraphPage 支持 URL query param `?card=xxx` 实现直接跳转

**路由:**
- `web/src/App.tsx`: 新增 `/graph` 路由

### 4. GraphExplorer & GraphNavigationPanel 更新

- EDGE_TYPE_LABEL_KEY/SORT/COLOR 映射表同步到 v3.7 ontology
- 移除 `mentions`；新增 `has_tag`, `in_section`, `contains`, `includes`, `belongs_to_topic`, `mentions_candidate`, `resolves_to`

### 5. i18n

- 新增 node type 键: `graph.node_type_card`, `graph.node_type_community`, `graph.node_type_topic`, `graph.node_type_entity`, `graph.node_type_concept_candidate`
- 新增 edge type 键: `graph.has_tag`, `graph.in_section`, `graph.contains`, `graph.includes`, `graph.belongs_to_topic`, `graph.mentions_candidate`, `graph.resolves_to`
- 移除 `graph.mentions`, `graph.approval_state_of`
- 新增 `graph.fact_graph`, `graph.candidate_graph`, `graph.node_count`, `graph.edge_count`, `graph.open_graph_view`, `graph.back_to_library`
- 新增 `nav.graph`: "知识图谱"/"Knowledge Graph"
- 中英文双语完整覆盖

### 6. Product Copy Tests 更新

- `test_local_graph_views_are_visible_list_fallbacks_without_graph_libraries`: 
  更新为 v3.8 语义 — vis-network 现在是授权渲染引擎，不再列入 forbidden list；
  LocalGraphPreview 仍保持纯 HTML/CSS（不引入 canvas 渲染库）

## 安全性审计

- [x] 不读取 .env 或 secrets
- [x] 不调用真实 LLM、Cubox、Upstage
- [x] 不处理真实私人资料
- [x] 不写真实 Obsidian vault
- [x] 不做 RAG answering / embedding / vector DB
- [x] 不破坏 explicit approval / human_approved 语义
- [x] vis-network 是轻量级（~200KB min）渲染库，经 roadmap/spec 授权
- [x] 所有图数据来自确定性 DeterministicGraphBuilder，不依赖外部服务
