MindForge Uses a Layered Service Architecture with Explicit Approval Gates

The backend follows a four-layer pattern: API routers handle HTTP concerns, service layer contains business logic, strategy layer encapsulates algorithm choices, and the persistence layer manages data storage. All knowledge cards must pass through an explicit approval gate — AI generates drafts, but only human confirmation promotes them to the library.

REST API Follows Consistent Envelope Pattern with Semantic Status Codes

All endpoints return a uniform JSON envelope with success flag, data payload, error message, and optional metadata for pagination. The approval workflow is reflected in the API through dedicated status transitions — cards move from ai_draft through review to human_approved with full provenance tracking at each step.

Engineering Workflow Uses Spec-First Development with Gate Enforcement

All feature work begins with a specification document reviewed against architectural constraints. Implementation is tracked through implementation notes, and every change must pass defined gates — ruff linting, pytest suite, and frontend build — before merging to main. The workflow ensures no untracked changes enter the knowledge base.
