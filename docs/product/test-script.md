# First-Cycle Test Script

日期: 2026-05-28
用途: User Validation 测试脚本 — 给参与者的任务说明

---

## 参与者引导语 (观察者朗读)

> 这是一个帮助你整理个人知识的工具。我们想知道第一次使用的人能不能顺利上手。
>
> 接下来我会给你几个小任务。请一边操作一边说出你的想法 — 这不是考试，没有正确答案。
> 如果卡住了，告诉我你卡在哪里就好，我会记录但不会帮忙。
>
> 准备好了吗？打开浏览器，我会告诉你怎么开始。

---

## Task 1: 了解当前工作区 (3 min)

**引导语:**
> 你已经打开了 MindForge 的主页。花三分钟随便看看，告诉我你看到了什么，你觉得这个工具能做什么。

**观察要点:**
- 第一眼停留在哪里
- 是否注意到 QuickStartWizard
- 是否能理解卡片布局
- 是否能理解侧边栏导航

**完成判断:**
- 用户说出了至少 2 个功能或模块名称

---

## Task 2: 创建自己的工作区 (3 min)

**引导语:**
> 现在点击 "开始使用" 或 "Create Demo Workspace"，按照提示完成初始设置。
> 创建完成后，看看你的知识库里面有什么。

**观察要点:**
- 是否能独立完成 wizard 三步
- 是否能理解 demo 知识卡片
- 是否能区分 demo 卡片和真实卡片
- 是否注意到 explicit approval 概念

**完成判断:**
- 成功创建 demo workspace 并看到 Library 中的卡片

**期望路径:**
1. 点击 QuickStartWizard → "开始使用"
2. Step 1: 了解 MindForge → 下一步
3. Step 2: 创建第一个工作区 → 点击 "Create Demo Workspace"
4. Step 3: 探索 → 进入 Library 查看卡片

---

## Task 3: 导入一段内容并审阅 (4 min)

**引导语:**
> 现在我想让你导入一段文字，让工具帮你分析和整理。
>
> 你的笔记内容是: "今天和产品团队讨论了用户反馈系统，我们决定在下一个版本中加入 NPS 评分和用户行为追踪。技术上需要前端埋点和后端存储。优先级 P1，预计两周完成。"
>
> 你可以用 Markdown 文件导入，或者直接粘贴。导入后处理它，然后去 Review 页看看。

**观察要点:**
- 是否能找到导入入口 (Sources 页面)
- 是否能理解 "处理" 的含义
- 是否能看到 ai_draft 卡片
- 是否能理解 Review 页面的 "审阅" 和 "审批" 概念
- 是否能独立完成审批

**完成判断:**
- 成功导入一段内容 → 触发 AI 处理 → 在 Review 页面看到 draft → 执行审批

**期望路径:**
1. 进入 Sources → Add Source (Markdown 或 Paste)
2. 输入笔记内容
3. 点击 Process Now
4. 收到处理完成提示
5. 进入 Review → 查看 draft 卡片
6. 点击 Approve

---

## Task 4: 检索刚才的知识 (3 min)

**引导语:**
> 现在试试在 Recall 页面搜索你刚才导入的内容。输入 "NPS" 看看能找到什么。

**观察要点:**
- 是否能找到 Recall 页面
- 是否能理解搜索框
- 搜索结果是否符合预期
- 是否注意到 explain 面板（搜索命中的原因说明）

**完成判断:**
- 搜索 "NPS" 找到了刚才审批的卡片

**期望路径:**
1. 进入 Recall
2. 搜索框输入 "NPS"
3. 看到包含 NPS 的卡片
4. 可选: 点开 explain 面板看为什么命中

---

## Task 5: 生成一篇总结 (3 min)

**引导语:**
> 最后，让工具基于你的知识库生成一篇 Wiki 总结。在 Wiki 页面生成后，导出到本地文件看看。

**观察要点:**
- 是否能找到 Wiki 页面
- 是否能触发生成
- 生成速度是否可接受
- 是否能找到 Export 页面并下载

**完成判断:**
- Wiki 生成成功 + 导出下载了文件

**期望路径:**
1. 进入 Wiki → 点击 "生成 Wiki"
2. 看到生成的 Wiki 内容
3. 进入 Export → 选择格式 (Markdown/ZIP)
4. 点击下载

---

## 任务后反馈

任务完成（或 25 分钟后），引导用户填写 Feedback Form (`docs/product/feedback-form.md`)。

---

## 备注

- 如果用户在任何 task 超过 2 倍预期时间，标记为 blocked 并跳到下一个 task
- 如果用户放弃某个 task，不催促，记录后继续
- 不暗示正确做法，不提示功能位置
- 用户如果在 Setup 页面要求配置 API key，告知 "当前使用内置演示模式，无需配置"
