## 1. Problem Frame

### P2-05: Zero Frontend Test Coverage

当前 `web/src/` 下 0 个测试文件。项目有 14 个页面 + 多个组件，全部无自动化测试覆盖。前端质量问题只能通过 browser smoke 手动发现。

**Root cause:** 项目初始化时未配置前端测试框架。`@playwright/test` 已安装但用于 E2E（未启用），缺少组件/单元测试基础设施。

`pyproject.toml` 有 `[tool.pytest.ini_options]` 但无 `[tool.coverage]`。`pytest-cov>=5.0` 已在 dev dependencies 中但从未配置。无法量化测试覆盖率。

当前结构：4 个顶层 class/function 组：
- `WebConfigService` (39 methods): config CRUD + provider status + model/routing + env detection + validation
- `ConfigUpdateError`: simple error wrapper

提取候选：env/dotenv detection 逻辑（`_read_dotenv_presence`, `_find_dotenv`, `_parse_env_key`, `DotenvPresence`）自成一体，可提取为 `web_config_env.py`。

## 2. Implementation Units

### U1: vitest + happy-dom Infrastructure Setup

**Goal:** 配置前端组件测试的最小可用基础设施。

**Files:**
- Modify: `web/package.json` — 添加 vitest/happy-dom/@testing-library 依赖 + test 脚本
- Modify: `web/vite.config.ts` — 添加 vitest config (environment: happy-dom, include: src/**)
- New: `web/src/test/setup.ts` — test setup (if needed for happy-dom globals)

**Approach:**
1. `npm --prefix web install -D vitest happy-dom @testing-library/react @testing-library/jest-dom`
2. 在 `web/vite.config.ts` 中添加 `test` 配置块
3. 在 `web/package.json` 的 `scripts` 中新增 `"test": "vitest run"`
4. 验证 `npm --prefix web run test` 可运行（0 tests 通过 = exit 0）

**Dependency justification（依赖新增说明）:**
- `vitest`: ~3MB devDep, 与 Vite 原生集成, 零配置即可运行 TSX 测试, 是 Vite 生态的标准测试方案
- `happy-dom`: ~2MB devDep, 轻量 DOM 实现, 比 jsdom 更快更轻, 用于 vitest 的 browser-like environment
- `@testing-library/react`: ~500KB devDep, React 组件测试标准库, 不依赖实现细节
- `@testing-library/jest-dom`: ~100KB devDep, 语义化 DOM matchers (toBeInTheDocument, toHaveTextContent)

以上均为 devDependencies，不影响生产 bundle。总计 < 6MB，属于前端测试的最小可行组合。

**Verification:**
- `npm --prefix web run test` exit 0
- `npm --prefix web run build` 仍 exit 0（依赖不污染生产构建）

### U3: Python Coverage Configuration

**Goal:** 配置 [tool.coverage] 使 `pytest --cov` 可用。

**Files:**
- Modify: `pyproject.toml` — 新增 `[tool.coverage.run]` + `[tool.coverage.report]` 配置

**Approach:**
1. 在 pyproject.toml 中添加 coverage 配置：
   - `source = ["src/mindforge", "src/mindforge_web"]`

2. 运行 `python -m pytest --cov --cov-report=term-missing tests/ -q --tb=short` 验证可用
3. 不设强制阈值（配置基础设施，阈值设定是后续 policy decision）

**Verification:**
- `python -m pytest --cov --cov-report=term-missing tests/ -q --tb=short` exit 0，输出覆盖率报告

## 5. Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|------------|
| vitest/happy-dom 与现有 TypeScript 配置冲突 | Low | vitest 与 Vite 共享 tsconfig，已知兼容 |
| 新增依赖下载失败 | Low | npm registry 公开可用 |
| web_config_env 提取引入循环导入 | Low | env detection 不依赖 WebConfigService，单向导入 |
| 前端测试运行时间过长拖慢 CI | Low | vitest 默认极快，单组件测试 < 1s |
